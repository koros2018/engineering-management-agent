# 工程管理智能体 (EMA) - 开发规范

> 基于 Manus Agent 架构规范

## 核心Rule

### 技术标准
1. **可执行代码 > 工具调用**：能生成代码解决的不用预定义工具
2. **Sandbox隔离执行**：所有外部代码在隔离环境运行
3. **Multi-Agent编排**：Main-Agent统一调度，Sub-Agent扁平执行
4. **规划-执行-验证循环**：不是线性执行，而是循环验证

### 命名规范
- **Agent命名**：大驼峰（PascalCase），如 `SafetyComplianceAgent`
- **Tool命名**：小写下划线（snake_case），如 `blueprint_parser_tool`
- **模块命名**：小写下划线，如 `src/agent/`
- **变量命名**：小写下划线，类名大驼峰

### 提交规范
```
feat: 新功能
fix: 修复
docs: 文档
refactor: 重构
test: 测试
chore: 杂项
```

### Agent实现规范
1. 所有Sub-Agent继承 `BaseAgent`
2. 每个Agent有 `execute()` 和 `validate()` 方法
3. 所有输出包含 `confidence` 字段（0-1）
4. 失败重试最多2次
5. 每个Agent至少3个测试用例

### 状态汇报要求
每次状态汇报必须包含：
- 服务状态（API/进程）
- 功能进度（P0/P1/P2）
- 版本信息（当前版本 + 最近commit）
- 下一步计划
