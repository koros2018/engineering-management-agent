## 快速开始（3步）

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动服务

```bash
./manage.sh start
```

### 3. 打开浏览器

```bash
./manage.sh open
# 或直接访问: http://127.0.0.1:6189
```

## 环境要求

- **Python**: 3.10+
- **内存**: 最低 512MB，推荐 2GB+
- **磁盘**: 最低 500MB
- **Ollama** (可选): 本地AI推理，不装也能用云端模型

### 可选：Ollama 本地模型

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 下载模型
ollama pull deepseek-r1:8b
```

独立守护进程，每3分钟自动巡检：

```bash
# 查看状态
sudo systemctl status ema-supervisor

# 查看日志
sudo journalctl -u ema-supervisor -f

# 重启
sudo systemctl restart ema-supervisor
```

**巡检内容：**
1. 服务健康检查（API/UI/Ollama/NVIDIA）
2. LLM 透测（对话响应超时检测）
3. 会话锁死检测（>15min无响应自动告警）
4. 项目进度监督（未提交文件提醒）

**自动修复：**
- API 挂了 → 自动重启
- UI 挂了 → 自动重启

## Docker 部署（生产环境）

```bash
# 构建镜像
docker build -t ema:latest .

# 开发模式（API + UI）
docker-compose --profile dev up -d

# 生产模式（API + Nginx + UI + SSL）
docker-compose --profile prod up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f api
```

### SSL/TLS 配置

生产环境已内置 SSL 支持：
- 自动生成自签名证书（开发用）
- 支持自定义证书（替换 `certs/` 目录下的文件）
- 安全头：HSTS / X-Frame-Options / CSP 等

### 知识库 RAG

内置 ChromaDB 向量数据库，包含 21 条工程规范种子数据：
- 对话时自动检索相关规范（检索增强生成）
- 支持手动导入更多规范文档
- 向量嵌入使用本地 Embedding 或云端 API

### 主动智能推送

系统定时检查并主动推送通知：
- 里程碑到期检查（每日自动巡检）
- 订阅到期提醒（提前 7 天）
- 配额预警（使用量达 80%）
- 安全告警（异常登录检测）
- 批量处理完成通知

## 云平台部署方案

### 方案一：单云服务器（推荐试用阶段）

```
云服务器 (2核4G)
├── Docker Compose
│   ├── ema-api (6188)
│   ├── nginx (80/443)
│   └── UI 静态文件
└── 数据卷持久化
```

### 方案二：多服务分离（生产阶段）

```
云服务器A: API服务 (FastAPI + Uvicorn)
云服务器B: 前端UI (Nginx 静态文件)
云服务器C: AI推理 (Ollama / 云端API)
对象存储:   图纸文件 + 生成文档
```

## 常见问题

**Q: 端口被占用？**
```bash
# 查看占用
lsof -i :6188
lsof -i :6189

# 修改端口
./manage.sh stop
# 编辑 manage.sh 中的 API_PORT / UI_PORT
./manage.sh start
```

**Q: 测试失败？**
```bash
# 确认API在运行
curl http://127.0.0.1:6188/health

# 运行单测（不需要API）
python3 -m pytest tests/ -k "not E2E" -q

# 运行E2E测试（需要API）
python3 -m pytest tests/test_phase22_e2e.py -v
```

**Q: Ollama 模型下载慢？**
```bash
# 使用国内镜像
export OLLAMA_HOST=http://your-mirror:11434
ollama pull deepseek-r1:8b
```

**Q: 如何更新知识库？**
```bash
# 通过 API 导入新规范
curl -X POST http://127.0.0.1:6188/api/v1/knowledge/import \
  -H "Authorization: Bearer <token>" \
  -F "file=@new_spec.pdf"
```

**Q: Docker 生产环境如何配置 SSL 证书？**
```bash
# 将你的证书放入 certs/ 目录
cp your-cert.pem certs/fullchain.pem
cp your-key.pem certs/privkey.pem
# 重启服务
docker-compose restart nginx
```

**Q: 如何查看主动推送的通知？**
```bash
# 通知显示在 UI 顶部 🔔 图标
# 也可通过 API 获取：
curl http://127.0.0.1:6188/api/v1/notifications \
  -H "Authorization: Bearer <token>"
```
