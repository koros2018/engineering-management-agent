## 二、本机部署步骤

### Step 1: 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt-get update && sudo apt-get install -y nginx

# 验证
nginx -v
```

### Step 2: 生成 SSL 自签证书

```bash
sudo mkdir -p /etc/nginx/ssl

sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/ema.key \
  -out /etc/nginx/ssl/ema.crt \
  -subj "/CN=localhost"
```

### Step 3: 配置 Nginx 反向代理

```nginx
# /etc/nginx/sites-available/ema
server {
    listen 443 ssl http2;
    server_name localhost;

    ssl_certificate /etc/nginx/ssl/ema.crt;
    ssl_certificate_key /etc/nginx/ssl/ema.key;

    # 上传文件限制
    client_max_body_size 500M;

    # UI 静态文件
    location /ui/ {
        alias /mnt/d/OpenClawDataworkspace/Projects/engineering-management-agent/ui/;
        try_files $uri $uri/ /ui/index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:5188/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
        proxy_read_timeout 300s;
    }

    # Swagger docs
    location /docs {
        proxy_pass http://127.0.0.1:5188/docs;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:5188/openapi.json;
    }
}

# HTTP 跳转 HTTPS
server {
    listen 80;
    server_name localhost;
    return 301 https://$host$request_uri;
}
```

启用：
```bash
sudo ln -sf /etc/nginx/sites-available/ema /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### Step 4: 配置 systemd 服务

```ini
# /etc/systemd/system/ema-api.service
[Unit]
Description=EMA API Server
After=network.target

[Service]
Type=simple
User=kezhigang
WorkingDirectory=/mnt/d/OpenClawDataworkspace/Projects/engineering-management-agent
ExecStart=/usr/bin/python3 /mnt/d/OpenClawDataworkspace/Projects/engineering-management-agent/src/main.py --host 0.0.0.0 --port 5188
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable ema-api
sudo systemctl start ema-api
sudo systemctl status ema-api
```

### Step 5: 验证

```bash
# API 健康检查
curl -k https://localhost/health

# UI 访问
curl -k https://localhost/ui/index.html

# Swagger
curl -k https://localhost/docs
```

## 三、替代方案：Docker Compose 部署 (已有)

EMA 已有完整 `docker-compose.yml`:

```bash
cd /mnt/d/OpenClawDataworkspace/Projects/engineering-management-agent
docker-compose up -d
```

优势：无需安装 nginx/systemd，一键启动。但需要 Docker 环境。
