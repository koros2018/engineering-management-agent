# EMA 支付对接完整预案 v1.0

## 一、微信支付对接

### 1.1 前置条件

| 项目 | 来源 | 状态 |
|------|------|------|
| 微信商户号 (MCHID) | 微信支付商户平台 (pay.weixin.qq.com) | ⏸️ 待申请 |
| API v3 密钥 | 商户平台→API安全→设置APIv3密钥 | ⏸️ |
| 商户私钥 (pem) | 本地生成，公钥上传商户平台 | ⏸️ |
| 商户证书序列号 | 商户平台→API安全 | ⏸️ |
| AppID | 微信开放平台 (open.weixin.qq.com) | ⏸️ |

### 1.2 申请流程 (逐步操作)

**Step 1: 注册微信支付商户号**
1. 打开 https://pay.weixin.qq.com
2. 点击"接入微信支付"
3. 选择主体类型（企业/个体工商户）
4. 提交营业执照、法人身份证、对公账户
5. 审核通过后获取商户号（1-3个工作日）

**Step 2: 生成商户API证书**
```bash
# 在本地生成RSA2048私钥
openssl genrsa -out apiclient_key.pem 2048

# 导出公钥
openssl rsa -in apiclient_key.pem -pubout -out apiclient_pub.pem
```
将 `apiclient_pub.pem` 内容上传到微信商户平台→API安全→申请API证书

**Step 3: 设置环境变量**
```bash
# .env 文件
WECHAT_APP_ID=wxXXXXXXXXXXXX
WECHAT_MCH_ID=1234567890
WECHAT_API_V3_KEY=32位随机字符串
WECHAT_PRIVATE_KEY_PATH=/etc/ema/certs/apiclient_key.pem
WECHAT_CERT_SERIAL_NO=商户证书序列号
```

### 1.3 代码对接 (已有框架)

`src/payment_sdk.py` 已实现完整框架：
- `wechat_jsapi_order()` — JSAPI下单
- `wechat_h5_order()` — H5支付
- `wechat_native_order()` — Native扫码支付
- `wechat_refund()` — 退款
- `wechat_order_query()` — 查询订单
- 完整RSA256签名 (WECHATPAY2-SHA256-RSA2048)

**待验证:**
- [ ] 真实支付回调URL配置（生产域名+HTTPS）
- [ ] API v3 签名验证（已在SDK中实现）
- [ ] 退款到账测试

## 二、支付宝对接

### 2.1 前置条件

| 项目 | 来源 | 状态 |
|------|------|------|
| AppID (PID) | 支付宝开放平台 (open.alipay.com) | ⏸️ 待申请 |
| 商户私钥 | 本地生成，公钥上传支付宝 | ⏸️ |
| 支付宝公钥 | 支付宝开放平台下载 | ⏸️ |

### 2.2 申请流程

1. 打开 https://open.alipay.com
2. 登录后进入"开发中心"
3. 创建应用→选择"网页/移动应用"
4. 配置应用公钥
5. 签约"电脑网站支付"或"手机网站支付"产品

### 2.3 环境变量
```bash
ALIPAY_APP_ID=2021XXXXXXXXXXXX
ALIPAY_PRIVATE_KEY_PATH=/etc/ema/certs/alipay_private.pem
ALIPAY_PUBLIC_KEY_PATH=/etc/ema/certs/alipay_public.pem
ALIPAY_NOTIFY_URL=https://ema.example.com/api/v1/payment/alipay-callback
ALIPAY_RETURN_URL=https://ema.example.com/ui/index.html
```
