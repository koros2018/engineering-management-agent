## 五、发现的问题与修复

### 问题 1: 图纸上传超时 🔴→✅

**现象**: 280字节PDF / 18KB DXF 上传均超时（20s无响应）

**根因**: `/api/v1/upload/analyze` 通过 `TechRdAgent.run_with_retry(task)` 执行 `full_analysis` pipeline，该链路依次调用 5 个工具（parser→classifier→analyzer→extractor→optimizer），PDF parser 的 OCR 子进程 (`tesseract.js Node.js subprocess`) 阻塞 7 秒。

**修复**: 上传端点改为直接使用 `BlueprintParser.parse()`，PDF 默认 `disable_ocr=True`。

```python
# 修复前: Agent pipeline (5步, 每步有内部开销)
agent.run_with_retry(task)  # → 超时

# 修复后: 直接解析 (1步, ~0.02s)
parser = BlueprintParser()
parser.pdf_parser.use_ocr = False  # PDF禁用OCR加速
result = parser.parse(tmp_path)
```

**验证**: DXF 解析从超时→0.02秒。✅

### 问题 2: LLM 模型测试 422 🔴→✅

**现象**: `POST /api/v1/llm/test` 传 JSON `{"model":"xx"}` 返回 422

**根因**: 端点签名使用 `model_id: str = Body(...)`（FastAPI 解析为字符串 Body），不支持 `{"model":"xx"}` JSON 格式。

**修复**: 改为 `async def test_llm_model(request: Request)`，同时支持 JSON 和 Form：

```python
# 修复: 同时支持 {"model":"xx"} 和 Form model=xx
ct = request.headers.get("content-type", "")
if "json" in ct:
    body = await request.json()
    model_id = body.get("model", "")
else:
    form = await request.form()
    model_id = form.get("model", "")
```

**验证**: JSON/Form 双模式均正常。✅

### 问题 3: 订阅端点 Content-Type 不匹配 ⚠️

**现象**: `subscribe` 端点传 JSON `{"plan_id":"pro"}` 返回 422

**根因**: 端点使用 `Form(...)` 参数，需 `application/x-www-form-urlencoded`。

**结论**: 设计如此（兼容前端表单），API 文档已标注。✅
