## 二、归档方案

### 2.1 按日期分目录

```
docs/04-项目进展/
├── 2026-05-14/
│   ├── status-report-2026-05-14-1403.md
│   ├── status-report-2026-05-14-1431.md
│   ├── status-report-2026-05-14-1441.md
│   ├── file-path-audit-report-2026-05-14.md
│   └── fix-review-error-2026-05-14.md
├── 2026-05-13/
│   └── ...
├── 2026-05-12/
│   └── ...
└── PROGRESS_*.md (历史报告保留根目录)
```

### 2.2 归档规则

| 报告类型 | 归档位置 | 说明 |
|---------|---------|------|
| Cron 状态报告 | `YYYY-MM-DD/status-report-*.md` | 每 10 分钟自动生成 |
| 修复报告 | `YYYY-MM-DD/fix-*.md` | Bug 修复完成后 |
| 审计报告 | `YYYY-MM-DD/*-audit-*.md` | 专项审计完成后 |
| 历史报告 | 根目录 | 2026-05-13 之前的报告保持不变 |

### 2.3 归档脚本

```bash
#!/bin/bash
# 自动归档脚本：archive_reports.sh

DATE=$(date +%Y-%m-%d)
TARGET="docs/04-项目进展/$DATE"

# 创建目录
mkdir -p "$TARGET"

# 移动今日报告
mv docs/04-项目进展/status-report-$DATE-*.md "$TARGET/" 2>/dev/null
mv docs/04-项目进展/fix-$DATE-*.md "$TARGET/" 2>/dev/null
mv docs/04-项目进展/*-audit-$DATE-*.md "$TARGET/" 2>/dev/null

echo "✅ 已归档 $DATE 的报告到 $TARGET"
```

## 三、清理策略

### 3.1 周报归档

每周日将当周报告打包压缩：

```bash
# 周日执行
cd docs/04-项目进展/
tar -czf 2026-W20-reports.tar.gz 2026-05-13/ 2026-05-14/ ...
mv 2026-W20-reports.tar.gz ../99-归档/
```

### 3.2 月度清理

每月初删除上月的 Cron 状态报告（保留修复/审计报告）：

```bash
# 每月 1 号执行
find docs/04-项目进展/ -name "status-report-*.md" -mtime +30 -delete
```

## 四、查找指南

### 4.1 查找今日报告

```bash
# 查看今日报告列表
ls docs/04-项目进展/$(date +%Y-%m-%d)/
```

### 4.2 查找特定主题

```bash
# 搜索所有报告
grep -r "智能审图" docs/04-项目进展/ --include="*.md"

# 搜索今日报告
grep -r "智能审图" docs/04-项目进展/$(date +%Y-%m-%d)/ --include="*.md"
```

### 4.3 查找最新状态

```bash
# 查看最新状态报告
ls -lt docs/04-项目进展/*/status-report-*.md | head -1
```
