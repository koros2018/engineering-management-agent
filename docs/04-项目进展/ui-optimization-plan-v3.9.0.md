## 二、优化目标

> **管理员：使用Agent + 管理优化EMA**
> **租户：高效使用Agent**

### 2.1 设计原则

1. **角色分离** — 管理员和租户看到不同的侧边栏
2. **渐进式** — 租户界面简洁，管理员界面功能完整
3. **一致性** — 管理员界面是租户界面的超集（不重复造轮子）
4. **零学习成本** — 租户打开就能用，不需要知道"管理"概念

### 2.2 管理员 vs 租户对比

| 功能 | 租户(free/pro/enterprise) | 管理员(admin/super_admin) |
|------|--------------------------|---------------------------|
| 图纸审查/文档/预算 | ✅ | ✅ |
| Agent对话 | ✅ (按套餐) | ✅ (全部) |
| 历史记录 | ✅ 自己的 | ✅ 全局 |
| 性能监控 | ❌ | ✅ |
| 行为分析 | ❌ | ✅ |
| 用户管理 | ❌ | ✅ |
| 租户管理 | ❌ | ✅ |
| 模型配置 | ❌ | ✅ |
| 预警中心 | ❌ | ✅ |
| 自动报告 | ❌ | ✅ |
| 操作日志 | ❌ | ✅ |

## 四、实施计划

### Phase 25-A: 侧边栏角色分离（核心）

**时间估算：** 1轮
**变更文件：** `ui/index.html`

1. 新增 `isAdmin` computed（替换 `isSuperAdmin`）
2. 性能监控/行为分析加 `v-if="isAdmin"`
3. 新增 "⚙️ 系统管理中心" 侧边栏区域（仅管理员可见）
4. 将管理相关条目移入新区域
5. 端到端流水线/生命周期成本从 ⋯ 提升到工作台
6. 调整 CSS 分隔线和视觉层级

### Phase 25-B: 管理面板拆分（结构）

**时间估算：** 1轮
**变更文件：** `ui/index.html`

1. admin面板5个Tab → 5个独立侧边栏面板
   - 驾驶舱概览 → `showAdminOverview`
   - 用户管理 → `showAdminUsers`
   - 租户管理 → `showAdminTenants`
   - 模型配置 → `showAdminModels`
   - 反馈管理 → `showAdminFeedback`
2. 每个面板独立 toggle + 独立加载
3. 互斥逻辑：同组面板切换自动关闭前一个

### Phase 25-C: 全局测试与回归

**时间估算：** 1轮

1. 运行 200 测试用例确认无破坏
2. 前端 UI 冒烟测试 (admin/tenant 双角色)
3. API 端点验证
4. 性能监控/告警面板验证

### Phase 25-D: admin.html 整合或移除

**时间估算：** 0.5轮

1. 评估 admin.html 与 index.html 功能重叠度
2. 将 admin.html 独有功能迁移到 index.html
3. 移除 admin.html（如全部功能已覆盖）
